How this token calculator works

Large language models do not bill by word count. They bill by tokens, which are smaller chunks of text. This page estimates token usage from the prompt you paste, then applies the current published per-million-token rates for a selected model.

For English text, one token is often roughly 0.75 words. CJK text can behave differently, so this calculator uses a slightly different estimate for Chinese, Japanese, and Korean characters to avoid undercounting.

Why are output tokens usually more expensive?

Reading a prompt is cheaper than generating new text, so most providers price output tokens above input tokens. That is why a long completion can cost noticeably more than the prompt that triggered it.

Does this tool send my prompt to your server?

No. The prompt stays in your browser. This page does not require API keys and does not call model APIs to estimate cost.

Which model prices are included here?

This version includes current text-model pricing for OpenAI GPT-5.4 family models, Anthropic Claude Opus 4.6 / Sonnet 4.6 / Haiku 4.5, and Gemini 2.5 Pro / Flash / Flash-Lite standard text rates.

Good uses for this page

It is helpful for budgeting prompts before you code, checking whether a workflow should use a premium model or a cheaper fast model, and estimating what happens when you scale from one call to hundreds or thousands.
